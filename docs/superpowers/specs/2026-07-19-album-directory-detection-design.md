# 专辑目录识别设计

## 目标

在目录树扫描期间识别专辑目录。若一个目录的直接音频子文件中，超过一半的文件具有同一个、且与目录名相似的 `album` 标签，则：

- 该目录以专辑图标显示；
- 扫描完成后自动展开该目录；
- 目录仍保留原有的手动展开/收起交互。

## 范围与约束

- 只检查目录的直接子文件，不递归统计子目录中的音频文件。
- 仅支持现有格式：MP3、FLAC、M4A、M4B、MP4。
- 读取标签失败或缺少 `album` 标签的文件不参与候选专辑名的计数，但仍计入目录直接音频文件总数，因此不会提高判定比例。
- 目录树仍会过滤不含受支持音频文件的目录，并维持当前按目录优先、名称排序的行为。
- 专辑判定在既有后台目录扫描命令中完成，避免阻塞 GTK 主线程。

## 数据模型

在 `FileTreeNode::Directory` 增加 `is_album: bool` 字段，并在用于渲染的 `TreeRow` 中携带相同字段。

`FileTreeNode` 保持对目录树结构、专辑判定结果和路径的唯一所有权；界面层只根据 `TreeRow.is_album` 选择图标，不读取标签或重新计算判定。

单目录链合并时，合并后的可见节点继承其末端目录的专辑属性与路径，以保持显示名称、展开路径和专辑状态一致。

## 专辑判定流程

扫描每个目录时：

1. 枚举目录的直接子项。
2. 将直接子项中支持的音频文件计为 `audio_file_count`，并递归构建所有子树节点。
3. 对每个直接音频文件只读取轻量的 `album` 标签值；读取失败或不存在时忽略其候选值。
4. 对归一化后相同的非空专辑名进行计数。
5. 对每个候选专辑名计算目录名相似度。只有同时满足以下条件的候选才能认定目录为专辑：
   - `candidate_count * 2 > audio_file_count`，即严格超过直接音频文件的一半；
   - 相似度分数至少为 `0.70`。
6. 任一候选满足条件即将目录标记为专辑；若没有候选满足则为普通目录。

标签读取失败仅降低命中比例，不使整个扫描失败。目录不可读取仍沿用现有的扫描错误处理。

## 名称归一化与相似度评分

目录显示名和专辑标签使用相同的归一化：

- Unicode 小写；
- 保留 Unicode 字母和数字；
- 删除空白、标点与分隔符，例如 `-`、`_`、`/`、`.`、圆括号和方括号；
- 删除归一化前缀中的纯年份或编号，例如 `2024 - Album Name` 的 `2024`。

归一化后少于 3 个字符的名称只能通过完全相等匹配，避免短字符串造成误判。

相似度取下列规则中的最高值：

| 条件 | 分数 |
| --- | ---: |
| 归一化后完全一致 | 1.00 |
| 一方完整包含另一方 | 0.85 |
| 字符二元组（bigram）Jaccard 相似度 | 0.00–0.70（实际比率） |

二元组 Jaccard 值为两个名称的不同连续双字符集合的交集大小除以并集大小。完全相等与包含关系在计算二元组前优先返回固定分数。由于阈值为 `0.70`，只有大量连续字符重合的名称会通过回退匹配。

## 自动展开

目录扫描完成后，应用遍历最终构建的树，收集全部 `is_album` 目录的路径，并与根目录路径一起写入 `expanded_paths`。

收集来自最终树而不是扫描过程中的中间目录，以适配已有的单目录链合并逻辑。用户可在渲染后通过原有点击行为收起或重新展开这些目录。

## 界面行为

在树行渲染中：

- 普通目录继续使用当前文件夹图标；
- 专辑目录使用 GTK 图标主题中的专辑/音频专辑图标；
- 展开箭头、选中态、缩进与点击行为不变；
- 音频文件图标不变。

当找不到主题中的专辑图标时，GTK 图标主题会使用其标准回退显示，不影响目录树操作。

## 测试

为可独立测试的目录扫描与名称匹配逻辑增加单元测试：

1. 归一化忽略大小写、空格和常见分隔符。
2. 完全相等、包含关系与二元组重合评分。
3. 短名称仅允许精确匹配。
4. 超过一半的相同、相似专辑标签将目录标记为专辑。
5. 恰好一半不标记为专辑。
6. 标签名不相似、缺失标签和读取失败不标记为专辑。
7. 标记为专辑的目录会在树遍历中被收集为自动展开路径。
8. 单目录链合并后保留最终目录的专辑状态和路径。

验证命令：

```sh
cargo fmt --check
cargo clippy --all-targets -- -D warnings
cargo test
```
